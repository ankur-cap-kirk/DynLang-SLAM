Closing the Gap to SOTA — Method Survey & 2-Week Plan

Project: DynLang-SLAM  ·  Current: BONN person_tracking 20.32 cm, balloon 57.35 cm  ·  SOTA: 4.5 cm / 3.7 cm (DG-SLAM)  ·  Date: 2026-04-16

Written after sequential literature review of MonoGS, SplaTAM, DG-SLAM, DGS-SLAM, Gassidy, BDGS-SLAM, WildGS-SLAM, GLC-SLAM, LoopSplat, Splat-SLAM, KBGS-SLAM, LGSBA. Per-paper notes live in research/literature/.

1. Diagnosis recap

Across 10 hyperparameter configs we’ve already seen:

  • ATE on person_tracking is invariant at ~20 cm regardless of KF density, loss weights, map growth, pose init.
  • ATE builds up as a single discrete drift event at frame ~65, not smooth accumulation.
  • balloon is 2.8× worse than person_tracking, and the YOLO mask does nothing on balloon (balloon is not in COCO).

Diagnosis: the gap is architectural, not tunable. Three concrete deficits:

  1. Keyframe poses are frozen after tracking. A bad pose at frame 65 is never revisited.
  2. Dynamic masking is class-based only. Anything outside COCO moving classes leaks into the map.
  3. No tracking-failure detection. When the optimizer plateaus on a hard frame, the bad pose is committed silently.

3. Ranked recommendations for 2-week course timeline

A. Keyframe-window local bundle adjustment (MonoGS-style) impact: high effort: ~200 LoC

What: Un-freeze keyframe poses during mapping. Pick window W of 5–8 recent/covisible KFs. Jointly optimize {T_k : k ∈ W} ∪ G against the union of observations across those KFs, using the same photometric + depth loss already in our tracker + isotropic regularization on Gaussians.

Why this first: directly attacks our diagnosed failure — the frame-65 discrete drift event is un-fixable today because that KF’s pose is frozen the moment tracking returns. Local BA gives it a second chance.

# dynlang_slam/slam/mapper.py  (sketch)
def local_ba_step(self, window_kfs):
    pose_params = [kf.pose_param() for kf in window_kfs]  # 6-DoF Lie
    opt = torch.optim.Adam([
        {'params': pose_params,   'lr': 1e-3},
        {'params': self.map.gaussian_params(), 'lr': 1e-3},
    ])
    for it in range(N_BA_ITERS):
        loss = 0
        for kf in window_kfs:
            rgb_r, depth_r, alpha = self.map.render(kf.pose)
            mask = (alpha > 0.95) & kf.static_mask
            loss = loss + 0.9*l1(rgb_r, kf.rgb, mask) + 0.1*l1(depth_r, kf.depth, mask)
        loss = loss + lam_iso * isotropic_reg(self.map)
        opt.zero_grad(); loss.backward(); opt.step()

Hyperparameters to borrow from MonoGS: window size 8, λ_iso = 10, KF covisibility IoU threshold 0.9, tracker early-stop at pose update < 1e-4.

Expected ATE: -5 to -10 cm on person_tracking. (MonoGS/SplaTAM hit 3–5 cm on walking_xyz with this mechanism in place; we're giving up a lot by freezing poses.)

B. Depth-warp dynamic mask (DG-SLAM) impact: high for balloon effort: ~80 LoC

What: Class-agnostic dynamic detection by depth consistency. For each recent KF i, reproject its pixels into current frame j using estimated pose + depth, then compare reprojected depth to current frame depth. Pixels with depth disagreement above threshold are flagged dynamic. Intersect across a window of N=4 frames (to filter random noise), then UNION with the YOLO semantic mask.

# dynlang_slam/mask/depth_warp.py  (sketch)
def depth_warp_mask(D_i, D_j, T_ji, K, e_th=0.6):
    """Returns bool mask: True where pixel in frame j is inconsistent with frame i (=dynamic)."""
    H, W = D_j.shape
    # pixel grid in j
    ys, xs = torch.meshgrid(torch.arange(H), torch.arange(W), indexing='ij')
    p_j = torch.stack([xs, ys, torch.ones_like(xs)], dim=-1).float()  # (H,W,3)
    # unproject j -> 3D in j's frame using j's depth
    P_j = (torch.inverse(K) @ p_j.unsqueeze(-1)).squeeze(-1) * D_j.unsqueeze(-1)
    # transform to i's frame
    P_i = (T_ji.inverse()[:3, :3] @ P_j.unsqueeze(-1)).squeeze(-1) + T_ji.inverse()[:3, 3]
    # project into i
    p_i = (K @ P_i.unsqueeze(-1)).squeeze(-1)
    u_i, v_i, z_i = p_i[..., 0]/p_i[..., 2], p_i[..., 1]/p_i[..., 2], p_i[..., 2]
    D_i_at_pj = grid_sample(D_i, torch.stack([u_i, v_i], dim=-1))  # bilinear
    return (D_i_at_pj - D_j).abs() > e_th

def fused_motion_mask(current, kf_window, yolo_mask):
    warp_masks = [depth_warp_mask(kf.depth, current.depth, T_kf_cur, K) for kf in kf_window]
    static_across_window = ~torch.stack(warp_masks).all(dim=0)  # pixel static in ALL warps
    dynamic_from_warp = ~static_across_window
    return dynamic_from_warp | yolo_mask

Why second: directly attacks the balloon gap. YOLO emits nothing on balloon; depth-warp mask WILL fire because the balloon’s depth is inconsistent between consecutive keyframes.

Expected ATE: balloon 57 cm → <25 cm plausible. person_tracking effect is small (+16% in DG-SLAM's ablation on sequences where semantic already works).

C. Silhouette gate on tracker loss (SplaTAM) impact: medium effort: ~20 LoC

What: Before computing tracker photometric/depth loss, multiply residual by (alpha_rendered > 0.95).float(). Pixels that lie in un- or under-mapped regions contribute 0.

# dynlang_slam/slam/tracker.py  (modify the residual line)
alpha = render_alpha(pose, gaussians)  # already computed in renderer
valid = (alpha > 0.95) & static_mask
loss_rgb = l1(rgb_r[valid], rgb_obs[valid])
loss_dep = l1(dep_r[valid], dep_obs[valid])

Why include it: cheap, orthogonal, helps every frame — especially early frames where the map is sparse and residuals on unmapped pixels pull the tracker toward garbage.

Expected ATE: -1 to -3 cm.

D. Tracking-failure detection + retry from alternate inits impact: medium effort: ~100 LoC

What: After each tracker run, compare final photometric loss to a moving median/MAD of recent frames. If loss_final > median + 3*MAD, mark frame as suspected failure and retry from alternate inits: (1) identity, (2) last-KF pose, (3) velocity-init + small random rotation perturbations. Keep the solution with the lowest final loss.

Why: our failure mode is a discrete event, so we don't need drift correction — we need to not commit the bad pose in the first place. No 3DGS SLAM paper does this explicitly, but it's a standard feature-SLAM trick (ORB-SLAM's relocalization mode).

Expected ATE: directly attacks the frame-65 event — if it fires correctly on that frame, could remove the 14 cm drift jump. But it has recall/precision risk (false-positive retries waste time). Do this in week 2 if we still have a discrete drift signature after A+B+C.

E. Loop closure (GLC-SLAM / LoopSplat) skip for now

Replica ablation shows only +7% ATE improvement from loop closure alone (0.29 → 0.20 cm). BONN person_tracking / balloon are short, near-linear trajectories with few real loop opportunities. Implementation cost (NetVLAD, LM solver, keyframe-centric map deformation) is high. Defer.

F. DINOv2 uncertainty (WildGS-SLAM) skip unless time left over

Cleanest class-agnostic story in the literature, but adds a 300 MB feature backbone and an online-trained MLP. The depth-warp mask (B) gets ~70% of the benefit at 10% of the cost. Revisit only if A+B don't close enough of the gap.

5. Honest expected outcome

Stacking A + B + C is a plausibly 2×–3× improvement. A realistic target:

  • BONN person_tracking: 20 cm → 8–12 cm (not SOTA 4.5, but competitive).
  • BONN balloon: 57 cm → 15–25 cm (dramatic relative win, class-agnostic mask is the story).

What we won’t match: DROID-VO-based trackers (DG-SLAM, Splat-SLAM) bake in a pre-trained optical-flow pose estimator trained on millions of frames. That's the residual gap to SOTA — a 2-week project can't close it without adopting DROID.

What we will have: an ablation showing three independent architectural interventions each contributing measurable improvement, and a balloon-sequence story that beats the method's own language-grounded baseline by a large margin — a tight CVPR-style course-project narrative.
