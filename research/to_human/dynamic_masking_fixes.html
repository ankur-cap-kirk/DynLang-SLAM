Dynamic Masking: From 35cm to SOTA

DynLang-SLAM Research Report — BONN person_tracking sequence

The Problem

Our binary YOLO + temporal filter dynamic masking makes ATE worse:

ModeATE RMSETime/frame
Static (no masking)35.03 cm2.0s
Dynamic (YOLO mask)42.32 cm2.01s
SOTA: BDGS-SLAM4.03 cm—
SOTA: DG-SLAM4.73 cm—

Binary masking removes 11% of pixels → less tracking signal → noisier poses → worse ATE

Root Cause Analysis

  1. Signal loss: Removing pixels reduces photometric + depth gradients for pose optimization
  2. No geometric verification: YOLO masks are semantic-only — they don't correspond to actual motion
  3. Tracker already tolerates dynamics: L1 loss + coarse-to-fine is somewhat robust to outliers; aggressive masking hurts more
  4. No contamination cleanup: Dynamic pixels leak into Gaussians before detection starts, persisting forever

6 Concrete Fixes

FIX 1 HIGHEST IMPACT Depth Warp Mask (from DG-SLAM, NeurIPS 2024)

Project depth between keyframes and check for residuals. Moving objects physically change depth; static walls don't.

def compute_depth_warp_mask(depth_cur, depth_prev, pose_cur, pose_prev, K):
    # Unproject current pixels to 3D
    pts_3d = unproject(depth_cur, K)
    # Transform to previous frame
    T_rel = pose_prev.inverse() @ pose_cur
    pts_prev = transform(pts_3d, T_rel)
    # Project back and compare depth
    depth_sampled = sample_depth(depth_prev, project(pts_prev, K))
    residual = (pts_prev.z - depth_sampled).abs()
    return residual > 0.1 * depth_cur  # relative threshold

Why: This is the #1 contributor to DG-SLAM's 4.73cm result. Combined with semantic: final = depth_warp | yolo_mask

~50 lines in tracker.py | ~1-2 hours

FIX 2 HIGH IMPACT Soft Per-Pixel Weighting (from DGS-SLAM)

Replace binary mask with continuous weights based on photometric residual histogram. 90th percentile threshold with sigmoid falloff.

residual = (rendered_rgb - gt_rgb).abs().mean(dim=0)  # (H, W)
threshold = torch.quantile(residual, 0.90)
weights = 1.0 - torch.sigmoid(10 * (residual - threshold) / threshold)
loss = (weights * pixel_loss).mean()

Why: Dynamic pixels have high residual. Down-weight them smoothly instead of removing entirely → preserves tracking signal.

~30 lines in losses.py | ~30 min

FIX 3 HIGH IMPACT Constant-Velocity Pose Init

Use motion model extrapolation instead of identity for initial pose guess. Camera motion is smooth — better init = fewer iterations = less drift.

if len(pose_history) >= 2:
    delta = pose_history[-1] @ pose_history[-2].inverse()
    initial_pose = delta @ pose_history[-1]  # extrapolate

Why: Better initialization means fewer iterations to converge and less chance of local minima. DG-SLAM uses DROID-SLAM for this; constant-velocity is simpler.

~15 lines in tracker.py | ~15 min

FIX 4 MEDIUM IMPACT Accumulated Opacity Mask (from DG-SLAM Eq. 9)

Only track against pixels where Gaussians have high accumulated opacity (well-reconstructed regions).

opacity_map = rendered["opacity"]  # from splatting
reliable_mask = opacity_map > 0.95
mask = mask & reliable_mask if mask is not None else reliable_mask

Why: Low-opacity regions are poorly reconstructed (often dynamics or unseen areas). Tracking against them adds noise.

~5 lines in tracker.py | ~15 min

FIX 5 MEDIUM IMPACT Bayesian Per-Gaussian Belief (from BDGS-SLAM)

Each Gaussian maintains a dynamic probability β ∈ [0,1] with temporal decay. Dynamic Gaussians get their opacity suppressed: effective_opacity = opacity * (1 - β)

Why: Soft suppression instead of hard pruning. Gaussians briefly occluded by a person recover via decay. This is how BDGS-SLAM achieves 4.03cm.

~40 lines in gaussians.py | ~1-2 hours

FIX 6 EASY WIN Depth-Verified Semantic Mask

Only trust YOLO detections where depth actually changed between frames. Stationary objects that YOLO calls "dynamic" get unmasked.

for det in detections:
    depth_diff = (cur_depth[det["mask"]] - prev_depth[det["mask"]]).abs().mean()
    if depth_diff < 0.05:  # less than 5cm = not actually moving
        det["mask"][:] = False

Why: Prevents false positives from stationary people, furniture misclassified as dynamic, etc.

~10 lines in pipeline.py | ~10 min
